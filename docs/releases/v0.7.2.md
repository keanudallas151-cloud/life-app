# v0.7.2 — notification system cleanup and Firebase sync reliability

## Summary

`v0.7.2` focuses on product reliability and app-shell consistency rather than another large feature drop. This release tightens Firebase cloud restore behavior, turns the notification template work into live runtime behavior, and surfaces the support contact directly inside the product.

## Included in this release

- Fixed `userData` cloud restore so existing Firestore documents load correctly whether they use camelCase or snake_case field names.
- Added shared notification helpers for:
  - default notification generation
  - template-based notification creation
  - relative timestamp formatting
  - local persistence
  - unread counting
  - mark-read and delete flows
- Replaced duplicated notification handling in:
  - `src/App.jsx`
  - `src/components/BottomNav.jsx`
  - `src/components/InventorsInvestors.jsx`
- Added runtime notification templates and started using them for:
  - profile updated
  - networking match / interest activity
  - shell defaults such as welcome and tailoring reminders
- Added a shared support-email config and surfaced it in Help and privacy copy.

## Why this matters

- Cloud-synced user data is less likely to appear “missing” after refresh.
- Notification behavior is now easier to extend without duplicating storage logic across multiple components.
- The app’s communication/template work is no longer documentation-only; parts of it now shape real runtime behavior.
- Users can see where to contact support directly inside the app.
